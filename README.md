# GBA Docker Template

Template repository for building Game Boy Advance homebrew projects with `devkitARM` in containers.

It provides three build images and a Makefile workflow that mounts `./source` into the container and runs your build command there.

## Images

- `base` (`dkarm_base:local`): `devkitARM` + core build tools
- `dusk` (`dkarm_dusk:local`): base image + extra utilities
- `butano` (`dkarm_butano:local`): dusk image + Butano engine

## Repository Layout

- `Makefile`: top-level container build/compile targets
- `docker/`: Dockerfiles for each image flavor
- `source/`: your GBA project (includes `source/Makefile` and `source/src/main.cpp`)

## Quick Start

1. Build the image you want:

```sh
make docker-build-butano
```

2. Build the ROM from inside the container:

```sh
make compile-butano CMD="make -j4"
```

3. Output:

- `source/source.gba` is generated by the project build
- top-level `./GBA_docker_template.gba` is copied automatically by `compile-butano`

## Common Commands

Build images:

```sh
make docker-build-base
make docker-build-dusk
make docker-build-butano
```

Run custom commands in containers:

```sh
make compile-base CMD="bash"
make compile-dusk CMD="make clean && make"
make compile-butano CMD="make -j4"
```

Clean Docker stamp cache:

```sh
make clean-docker-stamps
```

## Runtime Behavior

Before build/compile targets run, `check-container-runtime` verifies container runtime availability.

- Supports `docker` and `podman`
- Auto-starts Docker Desktop on:
  - macOS (via `open -a Docker`)
  - WSL2 Ubuntu (via `powershell.exe` / `cmd.exe`)
- Wait timeout can be adjusted:

```sh
make DOCKER_START_TIMEOUT=120 compile-base CMD="make"
```

You can force a runtime explicitly:

```sh
make CONTAINER_RUNTIME=podman compile-base CMD="make"
```

## Notes

- This template is based on `dkarm_gba_docker` and adapted for `GBA_docker_template` usage.
- Project-specific source changes should usually happen inside `source/`.
